Webinar-OMS-order managment system and state machine 12/18
State, transition, event, command
The order state machine is a computation model that describes the sequential logic of the (sales) order process.
There are five main concepts used in the Spryker order state machine:
State-describes the state of the process
Transition-forwards one state to another in a row
Event-determines a trigger for the transition
Condition-outlines a specific implementation of conditional decision logic
Command-defines extra functionalities added to an event

Testiranje, unit testovi i codeception
Testiranje na Sprykeru:
root codeception.yml file
Setup:
docker/sdk testing codecept generate:cest {ModuleName} {ClassName}
ovo generira prazni Codeception modul unutar pyz/tests
Najkorištenije naredbe:
vendor/bin/codecept build - generates base classes for all suites
vendor/bin/codecept run - runs all the test suites
Run tests: 
docker/sdk testing -x codecept run -c tests/PyzTest/Zed/{ModuleName}
docker/sdk testing codecept run -c tests/PyzTest/Zed/{ModuleName}
